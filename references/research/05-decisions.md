# Cos (余弦) / @evilcos 关键决策分析

## 一、创业决策：创办慢雾科技 (SlowMist)

### 决策背景
- **时间**: 2018年前后
- **行业背景**: 区块链行业快速发展，安全事件频发
- **个人状态**: 已在安全领域积累多年经验，从Web安全转向区块链安全

### 决策逻辑
> "我说我们在做区块链安全，其实我们在做都是公链生态的安全，慢雾的机会仅在这，如果这个生态没了，慢雾也会沉静甚至消失。是的，所谓的联盟链、私链不是我们的机会，我们的机会在自由的网络空间里:-)"

**核心洞察**:
1. **市场机会**: 区块链安全是一个新兴且快速增长的市场
2. **技术门槛**: 需要深厚的Web安全功底+区块链技术理解
3. **差异化定位**: 专注公链生态安全，而非联盟链/私链
4. **长期价值**: 区块链生态的安全是整个行业的基础设施

### 公司命名："慢雾"
- **"慢"**: "In security, slow is smooth, smooth is fast"
- **"雾"**: 神秘感，暗示安全的不可见性
- **哲学含义**: 安全需要耐心和细致，不能急于求成

### 决策结果
- 慢雾成为亚洲领先的区块链安全公司
- 产品线覆盖安全审计、威胁情报、AML反洗钱等
- 获得ISO/IEC 27001:2022认证
- 服务覆盖全球多个区块链项目

## 二、技术方向决策：从Web安全到区块链安全

### 决策路径
1. **早期**: XSS、CSRF、Web前端安全
2. **中期**: Flash攻击、浏览器安全、移动端安全
3. **后期**: 区块链安全、智能合约审计
4. **当前**: Web3安全、AI Agent安全

### 决策逻辑
- **技术演进**: 从攻击面的变化判断安全重心转移
- **市场趋势**: 区块链/Web3成为新的价值互联网
- **个人优势**: Web安全功底可以迁移到区块链安全

### 关键判断
> "区块链领域的黑客几乎都不是曾经的黑客，他们几乎都是很懂区块链的技术人员，且带了点逆向思维。曾经的黑客几乎都过时了，绝大多数没机会在这个新领域开天辟地，只能费劲地跟着。"

**洞察**: 传统黑客在区块链领域面临挑战，需要新的技术栈和思维方式。

## 三、开源决策：发布《区块链黑暗森林自救手册》

### 决策背景
- **时间**: 2022年前后开始持续更新
- **动机**: 教育用户，提升整个生态的安全水平
- **形式**: 开源文档，多语言版本

### 决策逻辑
1. **安全生态论**: 用户安全意识提升 → 生态更安全 → 慢雾受益
2. **品牌建设**: 高质量开源文档 → 行业权威性 → 商业机会
3. **社区贡献**: 开源精神，回馈社区
4. **长期价值**: 安全教育是长期投资

### 实施策略
- **版本迭代**: V1.0 → V1.1 → V1.2 → V1.3
- **多语言**: 中、英、日、韩、阿拉伯、印尼（6种语言）
- **持续更新**: 89次提交，最后更新2025年10月
- **扩展阅读**: darkhandbook仓库持续补充案例

### 决策结果
- 6.8k Stars，区块链安全领域最高star数的中文文档之一
- 建立了慢雾在安全教育领域的权威地位
- 成为区块链安全从业者的必读材料

## 四、工具开发决策：XSS'OR → Storage Inspector

### 决策逻辑
- **工具哲学**: "黑客工程化"——不只是发现漏洞，更要打造工具
- **用户需求**: 安全从业者需要实用工具
- **品牌建设**: 工具是技术能力的证明

### 演进路径
1. **XSS'OR (2017)**: XSS在线工具，"Hack with JavaScript"
2. **Storage Inspector (2025)**: Chrome扩展，检查localStorage/sessionStorage/Cookie
3. **vscode_tasks_command_execute_poc (2026)**: VSCode漏洞PoC

### 决策特点
- **实战导向**: 解决真实需求
- **开源优先**: 愿意分享工具
- **持续迭代**: 从XSS'OR到Storage Inspector的演进
